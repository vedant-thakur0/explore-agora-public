# CLAUDE.md — knowledge_graph/

## Knowledge graph-specific notes
- Prototype scope is Congress-only: `data/documents.csv` (622 docs), `data/segments.csv` (4,087 segments).
- Full corpus CSVs live one level up (`../documents.csv`, `../segments.csv`). Do not modify them.
- Join key: `documents.csv → AGORA ID` = filename in `data/fulltext/<agora_id>.txt`.
- Cosponsor data: `graph_data/agora_cosponsors_long.csv` (one row per document-cosponsor pair) and `graph_data/agora_comprehensive_data_with_cosponsor_lists.csv` (primary sponsor source).
- Community detection config (Louvain resolution, similarity weights) is in `pipeline/config.py`, not here.
- Graph outputs (`multiplex_graph/`) are regenerated by `pipeline/graph_builder.py` — do not hand-edit them.
- See `COSPONSOR_LAYERS.md` for Layer 1b (active cosponsor) and Layer 1.75 (withdrawn cosponsor) network details.
